# Weather MCP Server - 自动定位功能增强

## 修改概述

本次修改主要增强了weather-mcp-server的自动定位功能，使其更加可靠和用户友好。

## 主要改进

### 1. 多重定位服务支持

**原实现：**
- 仅使用ip-api.com一个定位服务
- 单一服务失败则无法获取位置

**新实现：**
- 支持4个定位服务，按优先级尝试：
  1. ip-api.com（中文）
  2. ip-api.com（英文）
  3. ipinfo.io
  4. ipapi.co
- 任一服务成功即可获取位置
- 失败时自动尝试下一个服务

### 2. 智能缓存机制

**新增功能：**
- 定位结果缓存1小时
- 避免频繁请求定位服务
- 提高响应速度
- 减少网络开销

### 3. 改进的错误处理

**增强内容：**
- 每个定位服务都有详细的错误日志
- 5秒超时设置，避免长时间等待
- 所有服务失败时提供清晰的错误提示
- 建议用户设置环境变量或手动指定城市

### 4. 工具描述优化

**改进：**
- 所有工具的`city`参数描述更新
- 明确说明"留空则通过IP自动定位获取当前位置"
- 让AI更清楚地理解自动定位功能

### 5. 编码问题修复

**修复：**
- 活动建议工具中的`tempC`字段错误
- 改为兼容`temp_C`和`tempC`两种字段名
- 确保在不同API响应格式下都能正常工作

## 技术细节

### 代码结构

```python
# 多重定位服务
location_services = [
    ("http://ip-api.com/json/?lang=zh", lambda d: d.get("city", "")),
    ("http://ip-api.com/json/?fields=city", lambda d: d.get("city", "")),
    ("https://ipinfo.io/json", lambda d: d.get("city", "")),
    ("https://ipapi.co/json/", lambda d: d.get("city", "")),
]

# 缓存机制
_DEFAULT_CITY_CACHE: str | None = None
_LOCATION_CACHE_TIME: float = 0
_CACHE_DURATION = 3600  # 缓存1小时
```

### 使用方式

#### 方式1：环境变量（推荐）
```bash
# Windows
set DEFAULT_CITY=北京

# Linux/macOS
export DEFAULT_CITY="北京"
```

#### 方式2：IP自动定位
- 不设置环境变量
- 系统自动通过IP定位
- 支持国内外网络环境

#### 方式3：手动指定
```json
{
  "city": "上海"
}
```
