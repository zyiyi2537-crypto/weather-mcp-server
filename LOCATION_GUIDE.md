# 定位方式说明

## 为什么IP定位不准确？

IP定位不准确的主要原因：

1. **运营商IP分配**
   - 中国电信、移动、联通等运营商的IP段分配可能与实际位置不符
   - 特别是移动网络，IP经常被分配到其他城市

2. **代理和VPN**
   - 使用代理或VPN时，IP显示的是代理服务器位置
   - 而不是你的实际位置

3. **IP数据库滞后**
   - IP定位服务的数据库更新不及时
   - 新分配的IP段可能没有及时更新

4. **动态IP**
   - 很多网络使用动态IP，位置信息经常变化

## 新的定位方案

### 1. GPS定位（最准确）

**精度：** 5-10米

**要求：**
- GPS硬件模块（USB GPS接收器、树莓派GPS帽等）
- 室外环境（需要接收卫星信号）
- 安装依赖：`pip install pyserial pynmea2`

**工作原理：**
1. 读取GPS模块的NMEA数据
2. 解析GPGGA语句获取经纬度
3. 使用OpenStreetMap反向地理编码获取城市

**优点：**
- 精度最高
- 不依赖网络
- 全球通用

**缺点：**
- 需要硬件
- 需要室外环境
- 首次定位需要时间

### 2. WiFi定位（较准确）

**精度：** 20-50米

**要求：**
- 连接到WiFi网络
- 管理员/root权限（用于读取WiFi BSSID）

**工作原理：**
1. 获取当前连接的WiFi BSSID（MAC地址）
2. 查询WiFi位置数据库
3. 获取经纬度并转换为城市

**优点：**
- 精度较高
- 室内可用
- 不需要额外硬件

**缺点：**
- 需要WiFi连接
- 需要管理员权限
- 依赖WiFi位置数据库

### 3. IP定位（备用方案）

**精度：** 城市级

**要求：**
- 网络连接

**工作原理：**
1. 查询多个IP定位服务
2. 获取IP对应的地理位置

**优点：**
- 无需特殊权限
- 无需额外硬件
- 随时可用

**缺点：**
- 精度低
- 可能不准确

## 定位优先级

系统按以下顺序尝试定位：

1. **环境变量** `DEFAULT_CITY`（最优先，用户手动设置）
2. **GPS定位**（最准确，需要硬件）
3. **WiFi定位**（较准确，需要WiFi）
4. **IP定位**（备用，精度最低）

## 如何选择定位方式

### 场景1：固定位置（推荐）
- 设置环境变量 `DEFAULT_CITY=你的城市`
- 最简单、最可靠

### 场景2：户外活动
- 使用GPS定位
- 需要GPS模块和室外环境

### 场景3：室内办公/家庭
- 使用WiFi定位
- 需要WiFi连接

### 场景4：移动办公
- 依赖WiFi或IP定位
- 可能需要频繁更新位置

## 常见问题

### Q: GPS定位失败怎么办？
A: 检查：
- GPS模块是否正确连接
- 是否在室外环境
- GPS模块是否已获取卫星信号
- 是否安装了pyserial和pynmea2

### Q: WiFi定位失败怎么办？
A: 检查：
- 是否连接到WiFi网络
- 是否有管理员权限
- 网络连接是否正常

### Q: 所有定位都失败怎么办？
A: 使用环境变量：
```bash
export DEFAULT_CITY="你的城市"
```

或在调用时手动指定城市：
```json
{
  "city": "北京"
}
```

## 总结

新的定位方案提供了多种定位方式，按优先级自动选择最准确的方式。对于大多数用户，推荐设置环境变量 `DEFAULT_CITY`，这是最简单可靠的方式。
